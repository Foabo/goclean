# goclean 使用示例

## 完整工作流程演示

### 1. 安装工具

```bash
# 使用 go install 安装
go install github.com/foabo/goclean@latest

# 验证安装
goclean -version
```

### 2. 基本使用

```bash
# 模拟运行，查看将要删除的模块
goclean -dry-run -verbose

# 实际清理（需要确认）
goclean -verbose
```

### 3. 指定扫描路径

```bash
# 扫描Go工作空间
goclean -modules ~/go -verbose

# 需要时扫描多个目录
goclean -modules ~/go,~/legacy-projects -dry-run
```

## 常见问题

### Q: 为什么没有找到未使用的模块？
A: 可能的原因：
- 您的项目使用了大部分缓存中的模块
- 扫描路径不正确
- 模块缓存目录为空

### Q: 删除模块后会影响现有项目吗？
A: 不会。工具只删除未被任何扫描项目使用的模块。如果项目需要，Go会重新下载。

### Q: 可以恢复被删除的模块吗？
A: 是的。当项目需要时，Go会自动重新下载被删除的模块。

### Q: 删除模块时遇到权限拒绝错误？
A: 这是因为Go模块缓存文件默认是只读的。解决方案：

**方案1（推荐）**: 工具现在会自动处理权限问题
```bash
goclean -verbose
```

**方案2**: 如果仍有权限错误，使用sudo
```bash
sudo goclean -verbose
```

**方案3**: 手动修复权限
```bash
chmod -R +w $(go env GOMODCACHE)
goclean -verbose
```

工具现在会自动尝试在删除前使文件可写，但某些系统可能仍需要管理员权限。
